= Bot de Trading - Manuel d'Utilisation

== Introduction

Ce bot de trading automatisé est conçu pour exécuter des stratégies de trading de cryptomonnaies systématiques sur plusieurs plateformes d'échange prises en charge. Le bot utilise une stratégie de moyenne d'achat progressive (DCA) combinée à des mécanismes de prise de bénéfices pour trader automatiquement les cryptomonnaies.

=== Plateformes d'Échange Prises en Charge
* *Binance* - Plateforme d'échange de cryptomonnaies mondiale
* *KuCoin* - Plateforme d'échange mondiale avec une sélection étendue d'altcoins
* *MEXC* - Plateforme mondiale de trading d'actifs numériques

[NOTE]
====
*Note* : La disponibilité des plateformes peut varier selon les régions. Assurez-vous que la plateforme choisie est accessible dans votre juridiction.
====

=== Fonctionnalités Clés
* *Trading Automatisé* : Achat et vente de cryptomonnaies basés sur des paramètres configurés
* *Gestion Multi-Positions* : Gestion simultanée de plusieurs positions d'achat
* *Optimisation des Bénéfices* : Marges bénéficiaires configurables et stratégies de réinvestissement
* *Mode Simulation* : Testez des stratégies sans trading réel
* *Surveillance en Temps Réel* : Suivez les performances du bot et les transactions en direct
* *Support Multi-Plateformes* : Exécutez des bots sur différentes plateformes simultanément

== Premiers Pas

=== 1. Inscription et Connexion

. Accédez à l'URL de l'application
. Sélectionnez votre plateforme d'échange préférée dans le menu déroulant
. Entrez vos identifiants (nom d'utilisateur et mot de passe)
. Cliquez sur *Connexion* pour accéder à la plateforme

[NOTE]
====
*Note* : L'inscription peut être désactivée selon la configuration du déploiement. Contactez votre administrateur si vous avez besoin d'un compte.
====

=== 2. Configuration de l'API de la Plateforme

Avant de créer des bots, vous devez configurer les clés API pour votre plateforme d'échange choisie :

. Accédez à la page *Compte*
. Trouvez la section *Plateformes d'Échange*
. Pour la plateforme sélectionnée, entrez :
+
* *Clé API* : Votre clé API de la plateforme
* *Secret API* : Votre secret API de la plateforme
* *Phrase Secrète API* : (Optionnel, requis pour KuCoin)

. Cliquez sur *Enregistrer* pour stocker vos identifiants

[NOTE]
====
*Note de Sécurité* : Les clés API sont chiffrées et stockées en toute sécurité. Ne partagez jamais vos clés API avec quiconque.
====

==== Permissions API de la Plateforme Requises
Vos clés API doivent avoir les permissions suivantes activées :
* *Lecture* - Consulter les soldes de compte et les données du marché
* *Trading* - Placer des ordres d'achat et de vente
* *Aucune Permission de Retrait Requise* - Pour la sécurité, les permissions de retrait doivent être désactivées

==== Comment Créer des Clés API

===== Binance

. Connectez-vous à votre https://www.binance.com[compte Binance]
. Cliquez sur l'icône de votre profil en haut à droite
. Sélectionnez *Gestion API* dans le menu déroulant
. Cliquez sur le bouton *Créer une API*
. Choisissez l'option de clé API *Générée par le Système*
. Entrez une étiquette pour votre clé API (par exemple, "Bot de Trading")
. Complétez la vérification de sécurité (2FA, email, etc.)
. Une fois créée, *copiez et sauvegardez immédiatement la Clé API et la Clé Secrète*
+
[WARNING]
====
La Clé Secrète n'est affichée qu'une seule fois ! Sauvegardez-la en toute sécurité avant de fermer la fenêtre.
====
. Cliquez sur *Modifier les restrictions* pour configurer les permissions :
+
* Activez *Activer le Trading Spot & Margin*
* Désactivez *Activer les Retraits* (pour la sécurité)
* Ajoutez l'adresse IP du bot à la liste blanche si nécessaire (IP affichée dans la page Compte)
. Cliquez sur *Enregistrer* pour appliquer les restrictions

===== KuCoin

. Cliquez sur le bouton *Créer une API*

* *Phrase Secrète API* : Créez une phrase secrète (vous en aurez besoin dans la configuration du bot)
* *Permissions* : Sélectionnez *Général* (permet le trading)
. Complétez la vérification de sécurité (2FA, email, etc.)
. Une fois créée, *copiez et sauvegardez immédiatement la Clé API, la Clé Secrète et la Phrase Secrète*
+
[WARNING]
====
La Clé Secrète et la Phrase Secrète ne sont affichées qu'une seule fois ! Sauvegardez-les en toute sécurité avant de fermer la fenêtre.

. Configurez les restrictions IP :
+
* Ajoutez l'adresse IP du bot à la liste blanche (IP affichée dans la page Compte)
* Cliquez sur *Confirmer* pour sauvegarder

[NOTE]
====
KuCoin nécessite les trois valeurs : Clé API, Secret API et Phrase Secrète API.

== Configuration du Bot

=== Créer un Nouveau Bot

. Accédez à *Bots* → *Nouveau Bot*
. Configurez tous les paramètres requis (détaillés ci-dessous)
. Cliquez sur *Créer Bot* pour initialiser
. Le bot sera créé en état de pause

=== Paramètres de Configuration Principaux

==== Paramètres de Base

*Étiquette du Bot* (Optionnel)
* Un nom personnalisé pour identifier facilement votre bot
* Exemple : "BTC Long-terme", "ETH Swing Trade"

*Intervalle du Bot* (Requis, 3-30 secondes)
* La fréquence à laquelle le bot vérifie les prix et exécute les trades
* Par défaut : 5 secondes
* Intervalles plus courts = plus réactif mais utilisation API plus élevée
* Intervalles plus longs = moins réactif mais utilisation API plus faible
* *Recommandation* : Utilisez 5-10 secondes pour la plupart des stratégies

*Paire de Trading* (Requis)
* La paire de cryptomonnaies à trader
* Format : `CRYPTO-USDT` (ex: BTC-USDT, ETH-USDT)
* Une fois définie, *ne peut pas être modifiée* pour un bot existant

==== Informations du Marché (Auto-remplies)

*Frais du Marché*
* Le pourcentage de frais de trading facturé par la plateforme
* Calculé automatiquement en fonction de votre niveau VIP (le cas échéant)
* Utilisé pour calculer des marges bénéficiaires précises

*Classe du Symbole*
* Classification de la paire de trading (uniquement sur certaines plateformes)
* Affecte la structure des frais

*Taille Minimale*
* Le montant minimum de cryptomonnaie pouvant être tradé
* Défini par la plateforme pour chaque paire de trading

*Incrément Minimum*
* Le plus petit incrément de prix pour la paire de trading
* Détermine la précision pour les ordres d'achat/vente

=== Paramètres d'Investissement

*Investissement Maximum* (Requis, minimum : 10 USDT)
* Le montant total d'USDT que vous souhaitez allouer à ce bot
* Exemple : 1000 USDT signifie que le bot utilisera jusqu'à 1000 USDT pour le trading
* Ce montant est divisé entre toutes les positions

*Positions Maximales* (Requis, minimum : 1)
* Le nombre maximum de positions d'achat simultanées que le bot peut détenir
* Exemple : 10 positions avec 1000 USDT = 100 USDT par position
* Plus de positions = meilleure moyenne mais tailles de position plus petites

*Prix par Position* (Calculé automatiquement)
* Le montant alloué par position
* Formule : `Investissement Maximum / Positions Maximales`
* Exemple : 1000 USDT / 10 positions = 100 USDT par position

*Positions Hautes* (Requis, par défaut : 1)
* Nombre de positions aux prix les plus élevés analysées pour déterminer la zone de prix élevés
* Utilisé pour éviter de racheter lors de la vente à des prix gonflés
* Plage : 1 à (Positions Maximales - 1)
* Valeurs plus élevées = zone de prix élevés plus large, stratégie de rachat plus conservatrice
* *Recommandation* : Commencez avec 1-2 pour un trading conservateur

*Bénéfices en Crypto* (Interrupteur)
* Lorsqu'activé, une partie de chaque vente conserve la cryptomonnaie au lieu de tout convertir en USDT
* Accumule la cryptomonnaie de base au fil du temps
* Exemple : Lors de la vente de BTC, conserver du BTC au lieu de tout convertir en USDT
* *Recommandation* : Activer si vous souhaitez accumuler la cryptomonnaie à long terme
* *Avertissement* : Assurez-vous que l'incrément minimum permet de petites quantités de crypto

[NOTE]
====
*Important* : La fonctionnalité "Bénéfices en Crypto" calcule la quantité de crypto à vendre pour atteindre votre objectif de profit USDT, puis vend le montant restant et conserve la différence en crypto.
====

*Réinvestir les Bénéfices jusqu'aux Positions Maximales* (Optionnel)
* Si défini, le bot ajoutera des positions supplémentaires en utilisant les bénéfices accumulés
* Exemple : Défini à 15 avec Positions Maximales de 10
+
** Le bot commence avec 10 positions
** Au fur et à mesure que les bénéfices s'accumulent, le bot peut croître jusqu'à 15 positions
** Une fois les bénéfices suffisants, les positions libres augmentent au-delà de 10

* Laisser vide pour désactiver cette fonctionnalité

*Réinvestir les Bénéfices* (Interrupteur)
* Lorsqu'activé, les bénéfices USDT sont utilisés pour augmenter la taille de chaque position
* Compose les bénéfices en rendant chaque nouvel achat plus important
* *Recommandation* : Activer pour les stratégies de composition à long terme

=== Paramètres de Stratégie de Trading

==== Comportement à la Baisse du Prix

Le bot offre deux modes pour configurer la réaction aux baisses de prix :

*1. Mode Seuil Unique* (Seuil constant)
* Un pourcentage fixe s'applique à toutes les positions
* Configuration la plus simple
* Exemple : 2% signifie acheter lorsque le prix baisse de 2% par rapport au dernier pic
* *Utiliser quand* : Vous voulez un comportement d'achat cohérent sur toutes les positions
* *Recommandation* : 1-3% pour les marchés volatils, 0.5-1% pour les marchés stables

*2. Mode Tableau de Seuils* (Profil de seuils)
* Définir différents seuils pour chaque position
* Permet une stratégie d'achat progressive à mesure que le prix baisse davantage
* Format : Valeurs séparées par des virgules (ex : `0.2, 0.2, 0.5, 0.5, 1.0`)
* Exemple avec 5 positions :
+
** Position 1 : Acheter quand le prix baisse de 0.2%
** Position 2 : Acheter quand le prix baisse de 0.2% supplémentaire (total 0.4%)
** Position 3 : Acheter quand le prix baisse de 0.5% supplémentaire (total 0.9%)
** Position 4 : Acheter quand le prix baisse de 0.5% supplémentaire (total 1.4%)
** Position 5 : Acheter quand le prix baisse de 1.0% supplémentaire (total 2.4%)

* *Utiliser quand* : Vous voulez acheter plus agressivement à mesure que le prix baisse davantage
* *Conseil Stratégique* : Commencez avec de petits seuils et augmentez-les pour les positions plus profondes
* *Note* : Le nombre de seuils doit correspondre ou dépasser vos positions maximales

*Choisir Entre les Modes :*
* *Seuil Unique* : Meilleur pour les débutants, comportement prévisible, moyenne uniforme
* *Tableau de Seuils* : Stratégie avancée, peut optimiser les points d'entrée, meilleur pour le DCA sur marchés volatils
* Vous pouvez basculer entre les modes à tout moment en modifiant la configuration du bot

==== Positions d'Urgence (Optionnel)

*Seuil de Position d'Urgence* (Optionnel, 0,1-100%)
* Débloque des positions d'urgence supplémentaires lorsque toutes les positions régulières sont épuisées
* Se déclenche lorsque le prix baisse de ce pourcentage par rapport au dernier prix d'achat
* Exemple : Seuil de 5% avec dernier achat à 50 000$ = Achat d'urgence à 47 500$

*Nombre de Positions d'Urgence* (Optionnel, défaut : 1)
* Limite le nombre maximum de positions d'urgence pouvant être actives simultanément
* Chaque position d'urgence fonctionne indépendamment avec son propre cycle vente/rachat
* Exemple : Définir à 3 permet jusqu'à 3 positions d'urgence concurrentes

*Comment ça fonctionne* :
+
** Lorsque toutes les positions sont utilisées (positions libres = 0)
** ET que le prix baisse du seuil d'urgence depuis le dernier achat
** Le bot achète une position d'urgence (si le solde le permet et limite non atteinte)
** La position d'urgence est automatiquement vendue et rachetée lorsque le prix augmente
** Crée un cycle de trading d'urgence auto-entretenu
** Si la limite est atteinte, aucune nouvelle position d'urgence ne sera achetée jusqu'à ce qu'une se vende

* *Cas d'usage* : Capturer des opportunités lors de fortes baisses de marché
* *Risque* : Utilise du capital supplémentaire au-delà de l'investissement maximum (investissement × positions d'urgence)
* *Recommandation* : Seuil 3-10%, Positions 1-3 selon la tolérance au risque et le capital disponible
* Laisser le seuil vide pour désactiver les positions d'urgence

[WARNING]
====
*Important* : Les positions d'urgence nécessitent un solde disponible au-delà de votre investissement maximum configuré. Assurez-vous d'avoir des fonds suffisants pour couvrir les achats d'urgence potentiels.
====

*Marge Bénéficiaire* (Requis, 0.1-100%)
* L'objectif de pourcentage de profit pour chaque position avant la vente
* Exemple : 1.5% signifie vendre lorsque le prix augmente de 1.5% au-dessus du prix d'achat (après frais)
* Inclut les frais d'achat et de vente dans le calcul
* Valeurs plus faibles = prise de bénéfices plus rapide, plus de transactions
* Valeurs plus élevées = prise de bénéfices plus lente, profits potentiellement plus élevés
* *Recommandation* : 0.8-2% selon la volatilité du marché

==== Évitement du Rachat en Zone de Prix Élevés

Le bot inclut une logique intelligente pour empêcher les rachats prématurés lors de la vente de positions à des prix élevés. Cette fonctionnalité aide à préserver le capital pendant les pics de prix et optimise la stratégie de rachat.

*Fonctionnement :*

Lorsque vous vendez une position, le bot rachète normalement immédiatement si des positions libres sont disponibles. Cependant, pour éviter de racheter à des prix gonflés, le bot analyse vos positions aux prix les plus élevés :

. Prend vos N meilleures positions (basé sur le paramètre "Positions Hautes")
. Identifie la plage de prix de ces positions les plus élevées
. Calcule une limite inférieure en utilisant le pourcentage de seuil
. Si le prix de vente se situe dans cette zone de prix élevés, le rachat est *ignoré*

*Détails de l'Algorithme :*
```
positionsHautes = Trier tous les achats par prix (plus élevé en premier)
                  Prendre les N premières positions (N = Positions Hautes)

prixMaxAchat = Prix d'achat le plus élevé
prixMin = Prix le plus bas parmi les N meilleures positions
limiteInférieure = prixMin × (1 - seuil/100)
prixVenteCible = prixMaxAchat × (1 + marge/100)
limiteSup = max(prixVenteCible, prixMaxTravail)

si (prix de vente est entre limiteInférieure et limiteSup) :
    Ignorer le rachat pour préserver les fonds pour de meilleurs points d'entrée
```

*Exemple Pratique :*

Configuration :
* Positions Maximum : 10
* Positions Hautes : 3
* Seuil : 2%
* Marge Bénéficiaire : 1.5%

Positions Actuelles :
```
Position 1 : 52 000 $
Position 2 : 51 500 $
Position 3 : 51 000 $
Position 4 : 49 000 $
Position 5 : 48 000 $
... (autres à des prix inférieurs)
```

Top 3 positions (N=3) :
* Prix max d'achat : 52 000 $
* Prix min : 51 000 $
* Limite inférieure : 51 000 $ × (1 - 0,02) = 49 980 $
* Prix cible vente : 52 000 $ × (1 + 0,015) = 52 780 $
* Prix max travail (si défini) : 53 000 $
* Limite supérieure : max(52 780 $, 53 000 $) = 53 000 $

Plage de Zone de Prix Élevés : 49 980 $ à 53 000 $

*Scénarios :*
* Vente à 51 500 $ : *Rachat ignoré* (dans la zone de prix élevés)
* Vente à 52 800 $ : *Rachat ignoré* (dans la zone de prix élevés, entre cible de vente et prix max travail)
* Vente à 53 500 $ : *Rachat autorisé* (au-dessus de la zone de prix élevés)
* Vente à 49 500 $ : *Rachat autorisé* (en dessous de la zone de prix élevés)

*Pourquoi C'est Important :*

* *Empêche l'Amplification des Pertes* : Évite de racheter immédiatement à des prix élevés pour les voir ensuite baisser
* *Optimise l'Efficacité du Capital* : Préserve les fonds pour de meilleurs points d'entrée à des prix plus bas
* *Réduit les Coûts de Transaction* : Moins de cycles achat-vente inutiles aux sommets
* *Améliore le Prix d'Entrée Moyen* : Attend les corrections de prix avant de racheter

*Conseils Stratégiques :*

* *"Positions Hautes" Plus Élevé* (3-5) : Zone de prix élevés plus large, stratégie de rachat plus conservatrice
* *"Positions Hautes" Plus Faible* (1-2) : Zone de prix élevés plus étroite, rachats plus agressifs
* *Marchés Volatils* : Utiliser des valeurs faibles (1-2) pour capturer les rebonds rapidement
* *Marchés Stables/en Range* : Utiliser des valeurs élevées (3-5) pour éviter les rachats prématurés

[NOTE]
====
*Note* : Cette fonctionnalité s'applique uniquement lorsque vous avez au moins autant de positions que votre paramètre "Positions Hautes". Avec moins de positions, la logique de rachat normale s'applique.
====

=== Plage de Prix de Travail (Optionnel mais Recommandé)

*Prix de Travail Minimum*
* Le prix le plus bas auquel le bot achètera
* Empêche l'achat lors de crashs sévères
* Laisser vide pour aucune limite minimale

*Prix de Travail Maximum*
* Le prix le plus élevé auquel le bot achètera
* Empêche l'achat aux sommets de prix
* Laisser vide pour aucune limite maximale

*Lors de la Création d'un Nouveau Bot :*

*% de Baisse Minimale* (Pour les nouveaux bots)
* Pourcentage en dessous du prix actuel pour définir le prix de travail minimum
* Exemple : 10% avec un prix actuel de 50 000 $ = Prix de travail minimum de 45 000 $

*% d'Augmentation Maximale* (Pour les nouveaux bots)
* Pourcentage au-dessus du prix actuel pour définir le prix de travail maximum
* Exemple : 10% avec un prix actuel de 50 000 $ = Prix de travail maximum de 55 000 $

[NOTE]
====
*Astuce Stratégique* : Définir une plage de travail aide à éviter de mauvaises décisions d'achat lors de conditions de marché extrêmes.
====

== Comportement du Bot

=== Logique de Trading

Le bot fonctionne selon une approche systématique avec la logique suivante :

==== Conditions d'Achat

Le bot exécutera un ACHAT lorsque *TOUTES* les conditions suivantes sont remplies :

. *Positions Libres Disponibles* : Au moins une position libre existe
. *Non Arrêté* : Le drapeau "Arrêter les Achats" n'est pas activé
. *Vérification de la Plage de Prix* : Le prix actuel est dans la plage de prix de travail min/max (si définie)
. *Un de Ces Déclencheurs* :
+
* *Premier Cycle* : Le bot vient de démarrer (cycle 0)
* *Baisse de Prix* : Le prix a baissé du seuil configuré par rapport au dernier prix le plus élevé
* *Rachat Après Vente* : Vient de vendre une position ET le nombre de positions rachetées est inférieur à la limite configurée

*Détails de l'Ordre d'Achat :*
* *Type d'Ordre* : Ordre au marché (s'exécute immédiatement au prix actuel du marché)
* *Montant* : Calculé à partir du prix de position divisé par le prix actuel
* *Arrondi* : Le montant est arrondi vers le bas pour respecter les exigences d'incrément de la plateforme

==== Conditions de Vente

Le bot exécutera une VENTE lorsque :

. *Prix Cible Atteint* : Le prix cible d'un achat a été atteint ou dépassé
. *Calcul du Prix Cible* :
+
[source]
----
Prix Cible = ((Payé + Frais d'Achat) × (1 + Taux de Frais de Vente) × (1 + Marge Bénéficiaire)) / Montant
----
+
Cela garantit que les frais et la marge bénéficiaire sont pris en compte.

*Détails de l'Ordre de Vente :*
* *Type d'Ordre* : Ordre au marché (s'exécute immédiatement au prix actuel du marché)
* *Montant Vendu* :
+
** Si "Bénéfices en Crypto" est *DÉSACTIVÉ* : Vend la position entière
** Si "Bénéfices en Crypto" est *ACTIVÉ* : Vend juste assez pour atteindre l'objectif de profit USDT, conserve le reste en crypto

==== Gestion des Positions

*Compteur de Positions Libres :*
* Commence à `Positions Maximales` lors de la création du bot
* Diminue de 1 à chaque achat
* Augmente de 1 à chaque vente
* Peut devenir négatif si des achats forcés manuels sont effectués
* Peut dépasser `Positions Maximales` si le réinvestissement des bénéfices est activé

*Suivi du Prix le Plus Élevé :*
* Le bot suit le prix le plus élevé atteint depuis la dernière vente
* Cette valeur est utilisée pour calculer quand un seuil de baisse de prix est atteint
* Se réinitialise au prix actuel après chaque achat

*Compteur de Positions Rachetées :*
* Suit combien de positions ont été rachetées pendant un marché en hausse
* Augmente lors de la vente et du rachat immédiat
* Empêche un rachat excessif pendant les hausses soutenues
* Se réinitialise lorsque le prix baisse et déclenche un achat basé sur la baisse

=== Cycle de Vie du Bot

==== 1. Création
* Le bot est créé avec la configuration
* État initial : *Non Démarré, Non En Pause*
* Le prix d'ouverture est enregistré

==== 2. Démarrage
* L'utilisateur clique sur le bouton "Démarrer"
* Le bot entre en état *Démarré, Non En Pause*
* La configuration est validée
* Les paires de trading et les clés API sont vérifiées
* Le bot commence sa boucle de cycle principale

==== 3. En Cours d'Exécution
* *Vérification des Prix* : Récupère le prix actuel à chaque intervalle
* *Évaluation des Conditions* : Vérifie les conditions d'achat et de vente
* *Exécution des Ordres* : Place des ordres d'achat/vente lorsque les conditions sont remplies
* *Persistance de l'État* : Enregistre l'état dans la base de données après chaque cycle
* *Journalisation* : Enregistre toutes les activités dans un fichier journal spécifique au bot

==== 4. Mise en Pause
* L'utilisateur clique sur le bouton "Pause"
* Le bot entre en état *Démarré, En Pause*
* Aucun nouvel ordre n'est placé
* Les positions actuelles restent ouvertes
* Peut être repris à tout moment

==== 5. Arrêt
* L'utilisateur arrête le bot (spécifique à l'implémentation)
* Le bot arrête de vérifier les prix et de placer des ordres
* Les positions restent ouvertes jusqu'à fermeture manuelle

=== Comportements Spéciaux

==== Modes Arrêt des Achats
Le bot dispose de deux contrôles indépendants pour arrêter les achats dans différentes conditions de marché :

*1. Arrêt des Achats lors de Baisse de Prix*
* Activé via le bouton "Arrêter les Achats en Baisse"
* Le bot cesse d'acheter lorsque les prix baissent mais continue le comportement normal de rachat lorsque les prix montent
* Utile quand vous croyez que le marché a touché le fond et que vous voulez préserver le capital
* Peut être inversé avec le bouton "Reprendre les Achats en Baisse"

*2. Arrêt des Achats lors de Rachat*
* Activé via le bouton "Arrêter les Achats au Rachat"
* Le bot cesse le rachat automatique après la vente de positions lors de la hausse des prix
* Continue à acheter lors de nouvelles baisses de prix selon la configuration
* Utile pour verrouiller les bénéfices lors de tendances haussières sans attraper les couteaux qui tombent
* Peut être inversé avec le bouton "Reprendre les Achats au Rachat"

*Les Deux Modes :*
* Le bot continue de vendre les positions rentables même lorsqu'un des modes est actif
* Peuvent être combinés (tous deux activés simultanément) pour mettre en pause toute activité d'achat
* Chacun peut être contrôlé indépendamment pour des scénarios de marché spécifiques

==== Actions Manuelles Forcées
Les utilisateurs peuvent déclencher manuellement des actions :
* *Vente Manuelle* : Vendre toutes les positions actuellement rentables
* *Achat Forcé* : Placer un ordre d'achat même si les conditions ne sont pas remplies
* *Vente Forcée* : Vendre des positions spécifiques même en dessous de l'objectif de profit
* *Mise à Jour de la Configuration* : Modifier les paramètres pendant que le bot fonctionne (champs limités)

==== Boost de Position
* Calculé automatiquement à partir des paramètres de réinvestissement des bénéfices
* Augmente les positions libres au-delà des `Positions Maximales` de base
* Formule basée sur le profit accumulé et le prix de position

==== Boost USDT
* Augmente le montant USDT par position lorsque "Réinvestir les Bénéfices" est activé
* Compose les bénéfices en positions plus importantes au fil du temps
* Calculé à partir du profit total divisé par les positions maximales

=== Stratégies de Profit

*Stratégie Conservative :*
* Marge bénéficiaire plus élevée (1.5-3%)
* Seuil de baisse de prix plus faible (0.5-1%)
* Moins de positions maximales (5-10)
* Pas de réinvestissement des bénéfices
* Résultat : Profits plus lents mais plus fiables

*Stratégie Agressive :*
* Marge bénéficiaire plus faible (0.5-1.2%)
* Seuil de baisse de prix plus élevé (2-5%)
* Plus de positions maximales (15-30)
* Réinvestissement des bénéfices activé
* Résultat : Profits plus rapides mais plus volatils

*Stratégie d'Accumulation :*
* Activer "Bénéfices en Crypto"
* Marge bénéficiaire moyenne (1-2%)
* Positions moyennes (10-20)
* Activer le réinvestissement des bénéfices
* Résultat : Accumuler de la cryptomonnaie tout en réalisant des profits USDT

== Dépannage

=== Problèmes Courants

==== Le Bot ne Démarre Pas

*Symptômes :*
* Le bot affiche "Non Démarré" après avoir cliqué sur Démarrer
* Messages d'erreur dans les journaux

*Solutions :*
. *Vérifier les Clés API* : Assurez-vous que les clés API sont correctement configurées
. *Vérifier les Permissions* : Les clés API doivent avoir des permissions de trading
. *Vérifier le Solde* : Assurez un solde USDT suffisant pour au moins une position
. *Vérifier la Paire de Trading* : Vérifiez que la paire de trading est disponible sur la plateforme
. *Vérifier les Journaux* : Consultez les journaux du bot pour des messages d'erreur spécifiques

==== Aucune Activité d'Achat

*Symptômes :*
* Le bot est en cours d'exécution mais ne place pas d'ordres d'achat

*Causes Possibles :*
. *Aucune Position Libre* : Toutes les positions sont remplies
+
** Solution : Attendez les ventes ou augmentez les positions maximales

. *Arrêt des Achats Actif* : Vérifiez si le mode "Arrêter les Achats en Baisse" ou "Arrêter les Achats au Rachat" est activé
+
** Solution : Cliquez sur "Reprendre les Achats en Baisse" et/ou "Reprendre les Achats au Rachat" selon le cas

. *Prix Hors de la Plage* : Le prix actuel est en dehors de la plage de travail
+
** Solution : Ajustez les prix de travail min/max

. *Solde Insuffisant* : Pas assez d'USDT pour une position
+
** Solution : Ajoutez des fonds ou réduisez la taille de position

. *Prix Pas Assez Baissé* : En attente du seuil de baisse de prix
+
** Solution : Attendez ou réduisez le seuil de baisse de prix

==== Aucune Activité de Vente

*Symptômes :*
* Le bot a des positions mais ne vend pas

*Causes Possibles :*
. *Prix Cible Non Atteint* : Le prix n'a pas assez augmenté pour le profit
+
** Solution : Attendez ou réduisez la marge bénéficiaire

. *Achats en Pause* : Certains achats peuvent être manuellement mis en pause
+
** Solution : Reprenez les achats en pause

. *Tendance Baissière du Marché* : Le prix est en dessous des prix d'achat
+
** Solution : Attendez la récupération du marché ou utilisez la vente forcée

==== Échec des Ordres

*Symptômes :*
* Messages d'erreur concernant les ordres échoués dans les journaux

*Causes Possibles :*
. *Solde Insuffisant* : Pas assez de fonds pour les ordres d'achat
. *Montant Trop Petit* : Taille d'ordre en dessous du minimum de la plateforme
+
** Solution : Augmentez la taille de position ou l'investissement maximum

. *Montant Trop Grand* : L'ordre dépasse le solde disponible
. *Limites de Taux API* : Trop de requêtes à la plateforme
+
** Solution : Augmentez l'intervalle du bot

. *Problèmes de Plateforme* : Problèmes temporaires de la plateforme
+
** Solution : Attendez et réessayez

==== Calculs de Profit Incorrects

*Symptômes :*
* Les profits ne correspondent pas aux attentes

*Explications :*
. *Frais* : Les frais d'achat et de vente réduisent le profit
. *Glissement* : Les ordres au marché peuvent s'exécuter à des prix légèrement différents
. *Bénéfices en Crypto* : Une partie du profit est conservée en crypto, réduisant le profit USDT
. *Arrondi* : Les petits montants peuvent être arrondis différemment

=== Messages d'Erreur

*"Clés API de plateforme invalides"*
* Réentrez vos clés API dans les paramètres du compte
* Vérifiez que les clés sont actives sur la plateforme

*"La paire de trading sélectionnée n'est pas disponible"*
* Vérifiez si la paire de trading existe sur votre plateforme
* Vérifiez le format de la paire de trading (ex: BTC-USDT)

*"Taille minimale non atteinte"*
* Augmentez l'investissement maximum ou réduisez les positions maximales
* Assurez que la taille de position respecte le minimum de la plateforme

*"Impossible de récupérer le prix actuel"*
* Vérifiez la connexion Internet
* Vérifiez que l'API de la plateforme est accessible
* Attendez la résolution des problèmes temporaires de la plateforme

*"Solde insuffisant"*
* Ajoutez plus d'USDT à votre compte de plateforme
* Réduisez la taille de position ou le nombre de positions

== Bonnes Pratiques

=== Gestion des Risques

. *Commencez Petit* : Commencez avec de petits investissements pour apprendre le système

. *Gardez des Réserves* : N'allouez pas 100% de votre solde de plateforme

=== Conseils de Configuration

. *Paramètres Conservateurs* : Commencez avec des marges bénéficiaires plus élevées et moins de positions
. *Plage de Travail* : Définissez toujours des prix de travail min/max pour éviter les achats aux extrêmes
. *Nombre de Positions* : Plus de positions = meilleure moyenne mais remplissage de position plus lent
. *Marge Bénéficiaire* : Couvrez au moins 2x les frais de trading (ex: 0.2% de frais = 0.5% de marge minimum)
. *Intervalle du Bot* : 5-10 secondes est optimal pour la plupart des scénarios

=== Sécurité

. *Clés API* : Ne partagez jamais vos clés API
. *Permissions de Retrait* : Désactivez les permissions de retrait sur les clés API
. *Restrictions IP* : Activez la liste blanche IP sur la plateforme si disponible
. *Audits Réguliers* : Vérifiez régulièrement les transactions du bot
. *Mots de Passe Forts* : Utilisez des mots de passe forts et uniques pour la plateforme

=== Maintenance

. *Vérifications Régulières* : Vérifiez les performances du bot hebdomadairement
. *Mises à Jour de Configuration* : Ajustez les paramètres en fonction des conditions du marché
. *Gestion du Solde* : Assurez un solde suffisant pour les opérations du bot
. *Vérification des Journaux* : Vérifiez les journaux pour les erreurs ou avertissements
. *Mises à Jour Logicielles* : Maintenez la plateforme à jour (contactez l'administrateur)

== Glossaire

*DCA (Dollar-Cost Averaging)* : Stratégie d'investissement consistant à acheter des montants fixes à intervalles réguliers

*Position* : Un seul ordre d'achat et son ordre de vente correspondant

*Positions Libres* : Nombre de positions disponibles pour l'achat

*Seuil de Baisse de Prix* : Pourcentage de baisse du prix avant l'achat

*Marge Bénéficiaire* : Pourcentage de profit cible pour chaque trade

*Plage de Travail* : Limites de prix min/max pour l'activité du bot

*Prix Cible* : Prix de vente calculé pour un achat afin d'atteindre la marge bénéficiaire

*Mode Simulation* : Mode de test où aucun trade réel n'est exécuté

*Frais de Plateforme* : Commission facturée par la plateforme pour chaque trade

*Ordre au Marché* : Ordre qui s'exécute immédiatement au prix actuel du marché

*USDT* : Tether, un stablecoin indexé sur le dollar américain

*Rachat* : Acheter à nouveau immédiatement après une vente lors d'une hausse de prix

*Action Forcée* : Remplacement manuel de l'automatisation du bot

*Glissement* : Différence entre le prix d'exécution attendu et réel
